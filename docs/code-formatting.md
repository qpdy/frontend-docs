# 代码格式化规范

## 缩进

- 统一使用 4 个空格进行缩进，不使用 Tab 字符
- 在 IDE 或编辑器中设置将 Tab 转换为空格

## 行宽

- 每行代码不超过 120 个字符
- 注释每行不超过 80 个字符

## 空行规则

1. 不同函数之间用一个空行分隔
2. 类中的方法之间用一个空行分隔
3. 逻辑代码块之间可以用空行分隔，提高可读性
4. 文件末尾保留一个空行

## 空格规则

### 运算符周围

- 二元运算符两侧添加空格，例如：`a = b + c`
- 逗号后面添加空格，例如：`function(a, b, c)`
- 冒号后添加空格，例如：`dict = {key: value}`

### 括号周围

- 关键字后的括号前不加空格，例如：`if(condition)`
- 函数调用的括号前不加空格，例如：`function()`
- 数组索引的括号前不加空格，例如：`arr[0]`

## 大括号位置

- 左大括号 `{` 放在行尾，右大括号 `}` 单独成行
- 示例：
  ```java
  if (condition) {
      // code here
  } else {
      // code here
  }
  ```

## 导入语句

- 按照标准库、第三方库、本地库的顺序分组导入
- 每组导入之间用空行分隔
- 同一组内的导入按字母顺序排列

## 注释格式

- 单行注释使用 `//` 或 `#`，并在符号后添加一个空格
- 多行注释使用 `/* */` 或 `''' '''`，内容与注释符号之间保持一个空格
