# 注释规范

## 注释的重要性

良好的注释能帮助他人和未来的自己快速理解代码的意图和实现逻辑，是高质量代码的重要组成部分。

## 注释类型

### 文件头注释

每个源代码文件都应该包含文件头注释，说明文件的基本信息：

```java
/**
 * 文件描述
 * 
 * @author 作者姓名
 * @version 版本信息
 * @since 创建日期
 */
```

### 行内注释

用于解释复杂或不直观的代码逻辑：

```java
// 解释为什么这样做，而不是代码做了什么
int result = calculateValue(); // 注意边界条件处理
```

## 注释语言

- 优先使用中文注释，确保团队成员都能理解
- 对于开源项目或国际化团队，可以考虑使用英文注释
- 保持注释语言的一致性

## 注释维护

- 修改代码时同步更新相关注释
- 删除无用或过时的注释
- 避免冗余注释，如 `i++; // i自增1`

## 特殊注释标记

- TODO: 标记需要后续完善的功能点
- FIXME: 标记需要修复的问题
- NOTE: 标记重要的注意事项
- HACK: 标记采用的特殊解决方案

示例：
```java
// TODO: 待优化算法复杂度
// FIXME: 此处存在内存泄漏风险
// NOTE: 此实现依赖特定版本的库
// HACK: 临时绕过框架限制
```
